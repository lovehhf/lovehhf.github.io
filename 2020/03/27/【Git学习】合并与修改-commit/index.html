推送新分支

由于 rebase 操作会修改版本树，如果合并的几个 commit 仅存在于本地，则直接 git push 推送即可

如果合并的几个 commit, 已经推送到远端了，则需要把本地的版本树覆盖掉远端，在 push 时使用 --force 或 -f 参数来强推

  • 要修改的 commit 是最新的一次提交
  • 要修改的 commit 不是最新的一次提交
